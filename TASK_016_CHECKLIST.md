# Task 016 — Ръчен чеклист (Codespaces)

Тези стъпки изискват реален човек: ти, реалния Telegram бот, и реален
разговор с Groq. `scripts/validate_task016.sh` покрива всичко останало
автоматично — пусни го първо.

## 0. Подготовка

В Codespaces secrets (или `.env`, никога в repото):

```
PHOENIX_TELEGRAM_BOT_TOKEN=<истински token>
AI_DEFAULT_PROVIDER=groq
GROQ_API_KEY=<истински ключ>
GROQ_MODEL=<провери текущ модел от console.groq.com/docs/models>
```

`PHOENIX_GITHUB_TOKEN` остави празен за момента — първо тестваме без
него (Task 016, т. 8: деградация без token).

## 1. Автоматична част

```bash
bash scripts/validate_task016.sh
```

Изчакай да завърши (~1-2 минути с реален pytest). Прочети
`task016_report.txt` — потвърди, че секции 1-8 казват "RESULT: ..." без
"FAILED"/"FOUND POTENTIAL LEAKS"/"⚠". Ако нещо гръмне тук, спри и ми
покажи `task016_report.txt`, преди да продължиш надолу.

## 2. Реален старт (за ръчните тестове)

Скриптът по-горе пуска приложението само за 8 секунди (за автоматичните
проверки). Сега го пусни за реално, в собствен терминал:

```bash
phoenix start
```

Остави този терминал отворен — ботът сега слуша реални Telegram
съобщения. Отвори нов терминал таб/прозорец за останалото (не спирай
този процес).

## 3. Реални Telegram команди

От телефона, в чата с твоя бот, изпрати **всяка** от тези, по ред, и
отбележи какво отговори:

- [ ] `/start` — очаквано: приветствие + версия
- [ ] `/help` — очаквано: списък с всички команди
- [ ] `/version` — очаквано: `🔥 Phoenix Core v0.1.0-alpha`
- [ ] `/status` — очаквано: ✅ за AI слой/Памет/AI Guard, ✅ за GitHub ако е конфигуриран (виж бележката за икона в TASK_015_REPORT.md)
- [ ] `/health` — очаквано: кратко `✅ Phoenix Core: healthy`
- [ ] `/ai` — очаквано: `groq: configured (по подразбиране)`
- [ ] `/memory` — очаквано: "Няма активен разговор" (все още няма `/ask`)
- [ ] `/reset` — очаквано: "Нямаше активен разговор"
- [ ] `/ask Hello` — **това е ключовият тест** — очаквано: реален отговор от Groq, форматиран като `🤖 Phoenix AI ... Provider: groq`

Ако `/ask Hello` върне грешка вместо истински отговор, копирай точния
текст на грешката тук и ми я покажи — там ще търсим причината.

## 4. Conversation memory на живо

- [ ] Изпрати `/ask Кажи ми едно число от 1 до 10`
- [ ] После `/ask Кое беше числото, което спомена?` — Groq трябва да
      помни предишния отговор (доказва, че conversation memory реално
      достига до истинския AI provider)
- [ ] `/memory` — трябва да покаже 4 съобщения (2x user + 2x assistant)

## 5. Rate limiting на живо (по избор, ако искаш да го видиш)

По подразбиране лимитът е 10 заявки/60 секунди — трудно се удря случайно.
Ако искаш реално да видиш rate limit съобщението:

```bash
# в Codespaces, временно, само за теста:
export AI_RATE_LIMIT_REQUESTS=2
# рестартирай phoenix start в неговия терминал (Ctrl+C, после пак `phoenix start`)
```

После изпрати `/ask` три пъти бързо — третия път трябва да получиш
"Твърде много заявки в момента." Не забравяй да махнеш
`AI_RATE_LIMIT_REQUESTS` override-а след теста (или просто рестартирай
терминала).

## 6. Реален restart тест

- [ ] В терминала с `phoenix start`, натисни `Ctrl+C` (graceful shutdown)
- [ ] Провери, че спира чисто (без grешки/traceback в изхода)
- [ ] Пусни пак `phoenix start`
- [ ] От Telegram: `/memory` — разговорът от преди рестарта трябва **все
      още да е там** (същия brой съобщения като преди спирането)

## 7. GitHub деградация (ако все още нямаш зададен token)

- [ ] `/repo` — очаквано: "GitHub клиентът не е конфигуриран" (не срив)
- [ ] `/issues` — същото очакване

Ако искаш да тестваш и с реален token, добави
`PHOENIX_GITHUB_TOKEN`/`PHOENIX_GITHUB_OWNER`/`PHOENIX_GITHUB_REPO`,
рестартирай, и пробвай пак `/repo`/`/issues` — очаквано: реални данни за
repото.

финалния Task 016 отчет с оценка READY / NOT READY, точно по формàта,
който заданието иска.
